AShooterGame

by: Ryan Trubey

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I wanted to create a shooter game that had different teams or sides. I also wanted to make a game where the player doesn't have to play as the same thing all the time, so the player can play as one of the two different teams.

2. What I Learned

I learned that different people prefer using different keys on the keyboard to control things in game. One suggestion that led to a change in my game was to make the spacebar the key to fire, because it was easier to use the spacebar to shoot when using the arrow keys to move.

3. Architecture of My Game

The first class is the runner class. This class is the only class with a main method and it runs the game. The second class is the GamePanel. This is a panel class where the game is displayed. This class also has all of the keyboard and mouse listeners. The game also has an ObjectManager class that spawns the enemies, updates the objects, and checks collisions. There is a player class that creates a player object with the player's position and if it is Human or Alien. There is also classes for the enemies and the projectiles. If the spacebar is pressed, the GamePanel class will call the shooting method from ObjectManager. In the shooting method, the objectManager creates a projectile object and adds them to a projectile array list at the player's location. The objectManager has a method to spawn the enemies, which creates enemy objects and adds them to an array list until it has reached the maximum amount of enemies. It is called in the actionPerformed method in the GamePanel every couple of seconds. The objectManager also has a method to check collisions. This method uses the collision boxes on all of the objects to check if they intersect and it is constantly being called in the actionPerformed method in the GamePanel class. The last method in the ObjectManager class cleans dead objects by checking they're isAlive variables. If they aren't alive, it removes them from the array list or ends the game if it is the player.

4. Technical Implementation

One aspect that I am proud of in my game is the buttons that I created with the mouse listener. I used if statements in the mouse listener to check which screen the game was on. Then, I used if statements to check if the mouse's x value was more than the button's left side and if the x value was less than the button's right side. I also checked if the mouse's y value was below the button's top side and above the button's lowest side. If all of these were true when the mouse was clicked, it did whatever the button was supposed to do when clicked.

5. Most Challenging Part

The most challenging part of this project was making the screen resizing option. The first challenge was figuring out how to resize the panel. After I figured out how to resize the game panel, there was another problem where if the player changed the screen size to something bigger than the screen, the game would be unplayable and they couldn't change the size back because the option went off of the screen. To fix this, I made it so that if the player tried to make the screen size bigger than the monitor's size, it would get the size of the monitor and set it to that size instead of making it what the user put.

6. Future Fixes and Updates

One improvement that I would like to make to the game is to make the size of everything in the game relative to the size of the screen, so that when the screen size becomes smaller, everything in the game also becomes smaller and when the screen becomes bigger everything gets bigger. If I were to continue working on the game, I would add levels and different abilities for the humans and the aliens to make the game more entertaining. I would also add the option to change the controls for the game to different keys on the keyboard or the mouse.
